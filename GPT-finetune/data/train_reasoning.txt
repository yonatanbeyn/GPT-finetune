Q: What is temperature in text generation?
<think>
Temperature T rescales logits before softmax: logit / T. When T < 1, the distribution becomes sharper (more peaked at the top token), making output more predictable and focused. When T > 1, the distribution flattens, increasing randomness and creativity. T = 1 is the unmodified distribution.
</think>
A: Temperature scaling controls the sharpness of the probability distribution during text generation. Lower temperature values make outputs more deterministic and focused, while higher values increase diversity and creativity by flattening the distribution over possible next tokens.

Q: What is top-p sampling?

</think>
A: Top-p sampling, also called nucleus sampling, restricts generation to the smallest set of tokens whose cumulative probability exceeds a threshold p. This technique balances quality and diversity by dynamically adjusting the candidate set based on the model's confidence at each step.

Q: What is dropout?

</think>
A: Dropout is a regularization technique where random neurons are set to zero during training with a specified probability. This forces the network to learn redundant representations, improving robustness and reducing overfitting without significantly slowing down training.

Q: What is causal masking?
<think>
Causal masking enforces the autoregressive property during training. In the attention matrix, positions are masked so that position i can only attend to positions 0 through i. Without this mask, the model could "cheat" during training by looking at future tokens. The mask is implemented as an upper-triangular matrix of negative infinity values added to the attention scores before softmax.
</think>
A: Causal masking in decoder-only transformers prevents each position from attending to future positions in the sequence. This ensures that during training and inference, the model only uses past context to predict the next token, maintaining the autoregressive property.

Q: What is beam search?
<think>

It tends to be more grammatically correct but less creative than sampling.
</think>
A: Beam search is a deterministic decoding algorithm that maintains multiple candidate sequences and selects the one with the highest overall probability. While it tends to produce high-quality outputs, it can be repetitive and lacks the diversity of sampling-based methods.

Q: What is learning rate warmup?

</think>
A: Warm-up scheduling gradually increases the learning rate from zero to its target value over an initial number of training steps. This technique stabilizes training in the early phases when gradients are large and the model is far from a good solution, preventing divergence.

Q: What is cosine annealing?

</think>
A: Cosine annealing is a learning rate scheduling strategy that decreases the learning rate following a cosine curve over the course of training. It allows the model to explore broadly with a high learning rate early on and then converge more precisely to a good minimum as training progresses.

A: The generalization gap refers to the difference in performance between a model on the training set and on unseen test data. Q: What is weight decay?
<think>
Weight decay adds a term to the loss: loss = task_loss + lambda * sum(w^2). This penalizes large weights, biasing the model toward simpler solutions with smaller weights. </think>
A: Weight decay is a regularization technique that adds a penalty proportional to the magnitude of model weights to the loss function. This discourages the model from learning unnecessarily large weights, helping to prevent overfitting and improve generalization on unseen data.

Q: What is Byte Pair Encoding?
<think>
BPE is a compression-inspired algorithm. Start with individual characters (or bytes). Count the frequency of all adjacent pairs. Merge the most frequent pair into a new symbol. Repeat until the desired vocabulary size is reached. The result is a vocabulary of common subword units that efficiently represents text with a fixed-size vocabulary.
</think>
A: Byte Pair Encoding is a subword tokenization algorithm that iteratively merges the most frequent pairs of characters or character sequences in a corpus. It produces a compact vocabulary that can represent rare and out-of-vocabulary words as sequences of subword units.

Q: What is a neural network?
<think>
A neural network is a stack of parameterized transformations. Each layer applies a linear transformation (matrix multiplication + bias) followed by a non-linear activation function. The non-linearity is crucial — without it, stacking linear layers produces only another linear transformation. Common activations include ReLU, GELU, and sigmoid.
</think>
A: A neural network is a computational model inspired by the structure of biological brains, consisting of layers of interconnected nodes called neurons. Each neuron applies a weighted sum of its inputs followed by a non-linear activation function, enabling the network to learn complex mappings from inputs to outputs.

Q: What is a loss function?
<think>
The loss function is the measure of how wrong the model's predictions are. It maps model outputs and true labels to a scalar value to be minimized. The choice of loss depends on the task: cross-entropy for classification/language modeling, MSE for regression, contrastive loss for similarity learning. The loss determines what the model learns to optimize.
</think>
A: A loss function measures how far the model's predictions are from the true targets during training. The choice of loss function depends on the task: cross-entropy is used for classification and language modeling, while mean squared error is common for regression problems.

Q: What is stochastic gradient descent?
<think>
SGD updates weights using a noisy estimate of the gradient computed from a mini-batch rather than the full dataset. This noise is actually beneficial: it prevents the optimizer from settling into sharp minima that don't generalize well. Full-batch gradient descent is more stable but computationally infeasible for large datasets.
</think>
A: Stochastic gradient descent updates model weights using the gradient computed from a single training example or a small mini-batch rather than the entire dataset. This introduces noise into the optimization process, which can help escape local minima and speeds up training significantly.

Q: How does self-attention work?
<think>
Self-attention operates in three steps per head: (1) project each token embedding into query, key, and value vectors using learned weight matrices, (2) compute attention scores = softmax(QK^T / sqrt(d_k)) — the dot product measures relevance between tokens, scaled to prevent vanishing gradients, (3) compute the output as a weighted sum of value vectors using the attention scores. The result lets each token "look at" all other tokens.
</think>
A: Self-attention computes a weighted sum of all token representations in a sequence, where the weights reflect how relevant each token is to every other token. For each position, it produces query, key, and value vectors, computes dot-product similarities between queries and keys, and uses the resulting weights to aggregate the values.

Q: What is a feed-forward network in a transformer?
<think>
Each transformer block has two sublayers: multi-head attention and a feed-forward network. The FFN is applied position-wise (same weights at every position). It typically has two linear layers with a GELU activation in between, and the hidden dimension is 4x the model dimension. It adds capacity beyond what attention provides.
</think>
A: Feed-forward networks within each transformer layer apply two linear transformations with a non-linear activation function in between. These sublayers learn to transform the attended representations into formats that are more useful for subsequent layers and the final prediction.

Q: What is the difference between GPT-2 and GPT-3?
<think>
The core difference is scale. GPT-2 has variants from 117M to 1.5B parameters. GPT-3 has 175B parameters — more than 100x larger. With this scale, GPT-3 demonstrates emergent abilities: reliable few-shot and zero-shot task completion through in-context learning, without any gradient updates. This was a qualitative shift in what language models could do.
</think>
A: GPT-3 is a much larger version of GPT-2, with 175 billion parameters compared to GPT-2's 1.5 billion. GPT-3 demonstrates significantly stronger few-shot and zero-shot capabilities due to its scale, and can perform many tasks without any fine-tuning through in-context learning alone.

Q: What is in-context learning?
<think>
In-context learning is striking because no weight updates occur. The model adapts to a new task purely by conditioning on examples in the prompt. Larger models are dramatically better at this. The examples appear to activate relevant "task representations" learned during pretraining. This is also why prompt engineering matters — the format of examples significantly affects performance.
</think>
A: In-context learning is the ability of large language models to perform new tasks by conditioning on a few examples provided directly in the input prompt, without updating any model weights. This emergent capability becomes more reliable as model size increases.

Q: What causes vanishing gradients?
<think>
Vanishing gradients arise from the chain rule of backpropagation. Gradients are multiplied through many layers. If those multiplications involve values consistently less than 1 (e.g., from sigmoid activations or certain weight matrices), the gradient signal shrinks exponentially with depth. By the time it reaches early layers, it is too small to drive learning.
</think>
A: Vanishing gradients occur when gradient signals become extremely small as they are backpropagated through many layers of a deep network. This prevents early layers from learning effectively. Techniques like residual connections, layer normalization, and careful weight initialization help mitigate this problem.

Q: What are residual connections?
<think>
A residual connection adds the layer's input directly to its output: output = layer(x) + x. During backpropagation, gradients flow through both the layer and the identity shortcut. The shortcut ensures gradients of at least magnitude 1 flow to earlier layers, solving the vanishing gradient problem for very deep networks like ResNet and transformers.
</think>
A: Residual connections, also called skip connections, add the input of a layer directly to its output before passing to the next layer. This allows gradients to flow more easily during backpropagation, enabling the training of very deep networks without vanishing gradients.

Q: What is the purpose of the feed-forward layer in a transformer?
<think>
After self-attention mixes information across positions, the feed-forward layer processes each position independently. It adds non-linearity and additional representational capacity. The 4x expansion in the hidden layer allows the model to express a richer set of functions than a simple projection. It is the largest contributor to transformer parameter count.
</think>
A: The feed-forward layer in each transformer block applies position-wise transformations to the representations produced by the attention mechanism. It typically expands the dimensionality by a factor of four before projecting back, allowing the model to learn complex non-linear feature transformations.

Q: What is greedy decoding?
<think>
Greedy decoding picks the highest-probability token at every step: argmax over the vocabulary. It is O(1) in memory and fast, but has a fatal flaw — a locally optimal choice at step t may not be globally optimal. For example, starting a sentence with a common word might lead to an awkward continuation, while a slightly less probable first word might lead to a much better sentence overall.
</think>
A: Greedy decoding selects the single most probable token at each generation step. It is fast and deterministic but often produces repetitive or suboptimal text because it does not consider the overall probability of the full generated sequence.

Q: What is model capacity?
<think>
Capacity is the expressive power of a model. More parameters means a larger hypothesis space — the model can represent more complex functions. But higher capacity requires more data to generalize well. The right capacity depends on the task complexity and data availability. Underfitting = too little capacity. Overfitting = too much capacity for the data available.
</think>
A: Model capacity refers to the ability of a model to learn complex patterns, determined by the number of parameters and the architecture. Higher capacity models can fit more complex data distributions but are also more prone to overfitting when training data is limited.

Q: What is a pretrained model?
<think>
Pretraining involves training on a very large general corpus to acquire broad knowledge. The model learns grammar, facts, reasoning patterns, and world knowledge. This learned representation then serves as a starting point for fine-tuning on specific tasks. The key insight: the pretrained weights are far better initialization than random weights for most downstream tasks.
</think>
A: A pretrained model has been trained on a large general dataset before being adapted to a specific task. Pretraining allows the model to learn broad representations of language or vision that can be efficiently transferred to downstream tasks with relatively little additional training data.

Q: What is weight initialization?
<think>
Random initialization must be done carefully. Too-large initial weights cause exploding gradients (activations saturate or grow unboundedly). Too-small initial weights cause vanishing gradients. Xavier initialization scales weights by 1/sqrt(fan_in + fan_out) for sigmoid/tanh activations. He initialization uses 2/fan_in for ReLU. These keep variance consistent across layers at initialization.
</think>
A: Weight initialization sets the starting values of model parameters before training begins. Random initialization with appropriate scaling, such as Xavier or He initialization, prevents vanishing or exploding gradients in the early stages of training deep networks.

Q: What is gradient clipping?
<think>
Exploding gradients are the opposite problem from vanishing: gradients grow exponentially through deep layers or over many time steps (common in RNNs). Gradient clipping caps the L2 norm of the gradient vector at a threshold. If the gradient norm exceeds the threshold, all gradients are rescaled proportionally. This keeps updates stable without changing their direction.
</think>
A: Gradient clipping limits the magnitude of gradients during backpropagation to prevent exploding gradients, which can destabilize training. By capping gradient norms at a threshold value, it allows stable training of deep networks and recurrent architectures.

Q: What is a mini-batch?
<think>
Mini-batch training is the practical middle ground. A mini-batch of 32-512 samples provides a noisy but useful gradient estimate. The noise helps regularize (acts like an implicit ensemble). The batch is small enough to fit in GPU memory and process quickly. Gradient accumulation can effectively simulate larger batches when memory is constrained.
</think>
A: A mini-batch is a small subset of the training data used to compute a single gradient update during training. Mini-batch gradient descent balances the efficiency of batch gradient descent with the regularization benefits of stochastic gradient descent.

Q: What is an autoregressive model?
<think>
Autoregressive generation is the process of generating one token at a time, each conditioned on all previously generated tokens. The joint probability of a sequence is factored as a product of conditional probabilities: P(x1, x2, ..., xn) = P(x1) * P(x2|x1) * P(x3|x1,x2) * ... This factorization is exact and the model is trained to maximize this likelihood.
</think>
A: An autoregressive model generates sequences one element at a time, conditioning each new element on all previously generated elements. GPT-2 and other causal language models are autoregressive, predicting each next token based on the full left context.

Q: What is the purpose of the end-of-sequence token?
<think>
Without an explicit stopping signal, the model would generate text indefinitely. The end-of-sequence token (EOS) is a special token in the vocabulary that the model learns to produce when a response is complete. During inference, generation halts when this token is sampled. During training, it is appended at the end of each example so the model learns when to stop.
</think>
A: The end-of-sequence token signals the model that a sequence is complete. During generation, the model stops producing new tokens when it outputs this special token, allowing it to generate sequences of variable length appropriate to the context.

Q: What is logit in the context of language models?
<think>
At each generation step, the model's final linear layer (the LM head) projects the last hidden state to a vector of size vocabulary_size. Each value in this vector is the raw unnormalized score (logit) for the corresponding vocabulary token. Softmax converts these logits to probabilities. Temperature divides the logits before softmax to control the output distribution sharpness.
</think>
A: A logit is the raw, unnormalized score output by the model's final linear layer for each token in the vocabulary. Logits are converted to probabilities using the softmax function, and the resulting distribution is used to sample or select the next token during generation.
